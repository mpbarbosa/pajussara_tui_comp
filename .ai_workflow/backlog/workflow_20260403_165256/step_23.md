## Detailed Findings & Recommendations

### 1. **Inefficient findIndex() in Hot Path**
**File:** `src/ListPanel.tsx`  
**Lines:** 54, 61, 74  
**Severity:** Medium  
**Impact:** For large `items` (hundreds+), repeated `entries.findIndex()` in render and effects causes O(n) scans per render, leading to UI lag.

**Example:**
```tsx
const idx = entries.findIndex((item) => item.id === selectedItemId);
```
**Occurs:**  
- On initial state  
- In useEffect  
- For scrolling window calculation

**Optimization:**
- Precompute a `Map<string, number>` of id → index once per items change.

**Before:**
```tsx
const idx = entries.findIndex((item) => item.id === selectedItemId);
```
**After:**
```tsx
const idToIndex = useMemo(() => {
  const map = new Map<string, number>();
  entries.forEach((item, i) => map.set(item.id, i));
  return map;
}, [entries]);
const idx = idToIndex.get(selectedItemId) ?? 0;
```

### 2. **Data Structure Choice: Record vs Array/Map**
**File:** `src/ListPanel.tsx`  
**Lines:** 38, 54

### 3. **No Memoization of Derived Data**
**File:** `src/ListPanel.tsx`  
**Lines:** 54, 74  
**Severity:** Low  
**Impact:** `entries`, `visible`, and `idToIndex` are recomputed on every render. For large lists, this is wasteful.

**Optimization:**  
- Use `useMemo` for all derived arrays/maps.

**Before:**
```tsx

```
**After:**
```tsx
const entries = useMemo(() => Object.values(items), [items]);
```
