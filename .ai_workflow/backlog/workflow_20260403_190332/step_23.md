## Detailed Findings

### 1. **src/ListPanel.tsx**
#### **Data Structure Choice: Array vs. Map**
- **Line(s):** Multiple (entries = Object.values(items))
- **Severity:** Medium
- **Impact:** For large lists, repeated `findIndex` calls on arrays are O(n) each. If `items` is large and selection changes frequently, this could add up.
- **Example:**
  ```tsx
  // Before
  const entries: ListItem[] = Object.values(items);
  const idx = entries.findIndex((item) => item.id === selectedItemId);

  // After (if items is large, keep a Map for O(1) lookup)
  const entries: ListItem[] = Object.values(items);
  const idToIndex = new Map(entries.map((item, i) => [item.id, i]));
  const idx = idToIndex.get(selectedItemId) ?? 0;
  ```
- **Note:** For small lists (as in demo), this is not a bottleneck, but for 1000+ items, consider a Map for selection/indexing.

### 2. **src/StreamViewer.tsx**
#### **Algorithmic Complexity: wrapText**
- **Line(s):** wrapText function
- **Severity:** Low
- **Impact:** The function slices the string in a loop, which is O(n) for n = text.length. For very large texts, this could be slow, but in TUI context, lines are short.
- **Example:**
  ```ts
  // Before
  while (remaining.length > 0) {
    lines.push(remaining.slice(0, maxWidth));
    remaining = remaining.slice(maxWidth);
  }
  // After (faster for very large strings)
  for (let i = 0; i < text.length; i += maxWidth) {
    lines.push(text.slice(i, i + maxWidth));
  }
  ```
- **Note:** Only relevant if liveText is very large (unlikely in TUI).

- **Line(s):** useRef for prevLiveText
- **Severity:** Low
- **Impact:** No leak, but if liveText is very large, could retain memory longer than needed. Not a practical issue here.
