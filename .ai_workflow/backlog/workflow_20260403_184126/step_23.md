### 1. **Algorithmic Complexity: O(n) findIndex in Hot Path**
**File:** `src/ListPanel.tsx`  
**Line(s):** Multiple (e.g., selection logic, visible window calculation)  
**Severity:** Low  
**Impact:** For small lists (typical in TUI), negligible. For 1000+ items, could cause sluggishness.

**Example:**
```tsx
const idx = entries.findIndex((item) => item.id === selectedItemId);
```
**Optimization:**  
If `items` is large and selection is frequent, maintain a `Map<id, index>` for O(1) lookup.

**Before:**
```tsx
const idx = entries.findIndex((item) => item.id === selectedItemId);
```
**After:**
```tsx
// Build once outside render:
const idToIndex = useMemo(() => {
  const map = new Map();
  entries.forEach((item, i) => map.set(item.id, i));
  return map;
}, [entries]);
// Use:
const idx = idToIndex.get(selectedItemId) ?? 0;
```

**Severity:** Low  
**Impact:** O(n) on string length. Acceptable for short AI output, but could be slow for very large outputs.

**Optimization:**  
If wrapping the same text repeatedly, memoize by input string and width.

**Before:**
```ts
export function wrapText(text: string, maxWidth: number): string[] {
  // ...
}
```
**After:**
```ts
const wrapText = (() => {
  const cache = new Map();
  return (text: string, maxWidth: number) => {
    const key = text + ':' + maxWidth;
    if (cache.has(key)) return cache.get(key);
    // ...wrap logic...
    cache.set(key, lines);
    return lines;
  };
})();
```
